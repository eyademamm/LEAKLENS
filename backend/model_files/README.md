# 📂 Model Files Directory

Place your trained model artifacts here **before starting the backend**.

## Required Files

| File | Description | Generated By |
|------|-------------|--------------|
| `water_leakage_model.keras` | Trained Conv1D Autoencoder | `another_copy_of_water_leakage_advanced.py` |
| `scaler.pkl` | Fitted `RobustScaler` | Same notebook |
| `threshold.pkl` | 95th-percentile MSE threshold (float) | Same notebook |

## How to Generate

Run the Colab notebook (`another_copy_of_water_leakage_advanced.py`) with your
`Final_SCADA.csv` dataset. At the end, it saves and downloads three files.

Copy them into **this folder**:

```
backend/
└── model_files/
    ├── water_leakage_model.keras   ← here
    ├── scaler.pkl                  ← here
    └── threshold.pkl               ← here
```

## Without Model Files (Mock Mode)

If the files are absent, the API automatically falls back to a **smart mock predictor** that:

- Detects **gradual pressure-drop patterns** → returns `is_anomaly: 1`
- Detects **stable low-variance data** → returns `is_anomaly: 0`
- Handles **random noisy data** → variable result

This lets the full web dashboard and mobile app function without a trained model.
